# Product Roadmap This roadmap sequences the two tracks that are being implemented together: 1. HighStakes-style AI quality checking. 2. Loop-engineering safety controls and loop discipline. The GTK-free desktop host migration plan lives in `docs/roadmap/gtk-free-host-migration-plan.html`. ## Roadmap Priority 1. Shared planning artifacts and traceability. 2. AI quality checking and PR gating. 3. Loop-engineering controls and run discipline. 4. Convergence and release hardening. 5. Release/security governance. ## Phase Plan | Stage | Primary Track | Scope | Dependencies | Exit Criteria | Bead Issues | |---|---|---|---|---|---| | Stage 0 | Shared | Create the backlog artifacts, phase order, tracker rows, and validation runner boundary. | None | `docs/feature-list.html`, `docs/product-roadmap.html`, and the bead tracker rows exist and cross-link each other, and the UI unit runner stays separate from the Playwright e2e suite. | BI-020, BI-038, BI-043 | | Stage 1 | AI quality checking | Add the versioned risk contract, admin command surface, file and PR scoring, CI gates, incident feedback, release baseline management, job observability, and browser bridge coverage. | Stage 0 | Risk output is deterministic, PR tiering is stable, the admin command surface is wired, browser smoke is authored, release baseline query/reseed controls are visible, job observability metrics are visible, and high/critical merges are blocked with a readable summary. | BI-021..BI-024, BI-030..BI-042 | | Stage 2 | Loop-engineering controls | Add triage formatting, budget guardrails, verifier separation, and minimal-fix escalation. | Stage 0, Stage 1 outputs | Report-only mode works, kill switch is enforced, verifier defaults to reject, fix proposals stay one-problem and cap retries, and run logs are append-only. | BI-025..BI-028 | | Stage 3 | Convergence | Merge both tracks into a single release gate and harden the shared workflow. | Stage 1, Stage 2 | All active work has test mapping, phase gates, and acceptance criteria; there are no unlabeled tasks.
