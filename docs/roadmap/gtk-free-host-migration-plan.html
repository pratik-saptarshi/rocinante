# GTK-Free Host Migration Plan ## Objective Retire the GTK/GLib-backed desktop host path without losing desktop parity. The long-term target is a GTK-free native shell with the same user-visible behavior, backed by a host-agnostic Rust core. ## Execution Model - Use TDD for every phase: write the failing test or contract first, implement the minimal code required to pass, then refactor only after green. - Use parallel agents for independent workstreams: - one agent maintains backlog and bead tracker consistency, - one agent writes and runs focused tests, - one agent keeps host and security docs synchronized. - Keep host adapters thin and preserve user-visible behavior while host internals change. - Treat browser-sidecar only as a scoped fallback for unresolved parity gaps, not as the default architecture. - Treat any active `gtk` or `glib` release-dependency path as a release-blocking parity failure. ## Replacement Strategy - Introduce `rocinante-core` as the host-agnostic behavior domain: - risk scoring, triage, verifier, budget guard, release baseline, and auth policy. - Introduce `rocinante-desktop-shell` as the host transport: - window lifecycle, shell event routing, tray/menus/shortcuts, - notifications, file pickers/dialogs, deep links, persistence restore, - updates and installation hooks. - Keep transport, IPC, and shell events behind thin platform adapters. - Keep browser-sidecar behind a separate adapter module so it is never the default host flow. - Remove GTK/GLib from first-party host dependencies and gate release on a clean dependency tree. ## Staged Feature Refinement Each phase is executed with explicit TDD + BDD gates. | Stage | Epic | Feature | for current desktop behavior. - Classify each item as must-have, should-have, or can-defer. - Compare native desktop, browser-sidecar, and temporary Tauri candidates. - Record signed host decision and fallback scope before any host rewrite. ### Phase 1: Core Extraction - Move domain behavior into `rocinante-core` with no GUI/runtime dependencies. - Define host-agnostic interfaces for all parity-critical flows. - Preserve behavior with red-green contract tests before host implementation begins. ### Phase 2: GTK-Free Native Desktop MVP - Implement baseline host behavior without GTK/GLib: - window lifecycle - menus and shortcuts - dialogs and file selection - tray and notification behavior - deep links and app launch flow - persistence and startup restoration - Keep command contracts and UX behavior stable. ### Phase 3: Parity Closure and Fallback Containment - Implement or explicitly defer every non-must-have item. - Keep any fallback adapter bounded and auditable. - Add closure tests so must-have items cannot regress and sidecar remains non-default. ### Phase 4: Remove Tauri/GTK/GLib - Remove old host crate/runtime wiring. - Remove GTK/GLib ignore coverage and update exception registry semantics. - Add dependency-tree + release docs gates that fail if `gtk` or `glib` returns. - Remove stale scripts and docs that assume GTK/Wry is the default path. ## removal, and publish/CI non-regression. ### Acceptance Gates by Stage - `Gate P0`: `T-056` fails until parity matrix has full required classes and host decision is signed. - `Gate P1`: `T-057` fails
