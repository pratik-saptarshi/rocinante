# Repository Security Baseline ## Main Branch Standards - Default branch: `main`. - Protected branch policy (configure on remote): - Require pull request before merge. - Require status checks (`CI`, `Security`) to pass. - Require linear history. - Block force pushes and branch deletion. ## Privacy Controls - Never commit secrets or credentials. - `.env*`, keys, certs, and runtime databases are ignored by default. - Secret scanning enabled in CI. ## Verification Controls - Mandatory CI checks: format, clippy, tests. - Scheduled security scans with CodeQL. - Dependency updates via Dependabot. - Open Dependabot alerts are tracked as release blockers until remediated or explicitly accepted. - Release-critical workflow actions and language toolchains must remain on supported stable releases; CI deprecation warnings on release jobs are treated as blockers until refreshed or formally accepted. - Dependency audit exceptions are tracked in `docs/roadmap/security-advisory-exceptions.json`. - `scripts/dependency-floor-proof.sh` records the current GTK/glib dependency floor. ## Release Blockers - `RT-RC-001` remains open while the GTK/glib dependency floor is present in `src-tauri/Cargo.lock`. - Release readiness is blocked by the tracked GTK/glib advisory exception. - Release readiness is blocked until the tracked GTK/glib advisory exception is retired or formally accepted. - `R3-F08` remains open until the UI lockfile resolves `esbuild` at `0.28.1` or newer and the remote Dependabot alert closes. - `RT-RC-003` remains open until rust-coverage no longer emits the Node.js 20 deprecation warning and the build/release freshness gate is green.
