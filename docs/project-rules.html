# Project Rules - No default exports. - Co-locate tests under a `/test` folder at the project parent level. - Use `snake_case` for database columns. - All documentation must use `.html` files under `/docs` or its subfolders. - Exception: `README.md` must remain in the project root. - Default branch must be `main`. - All changes to `main` must go through pull requests with passing `CI` and `Security` checks. - Never commit secrets, credentials, certificates, or private keys.
